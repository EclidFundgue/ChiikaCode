# ChiikaCode 用户手册

1.1 **编写目的**

本用户手册旨在帮助用户了解并正确使用 **ChiicaCode** 项目的所有功能。手册详细介绍了软件的安装、配置、使用方法以及注意事项，旨在为开发者提供一个高效、便捷的代码辅助工具。目标读者为所有使用 VSCode 进行开发的程序员，以及希望利用智能代码辅助工具提高工作效率的用户。

1.2 **项目背景**

**ChiicaCode** 是由 **Chiikawa 开发组** 自主开发的智能代码辅助工具，项目起源于提高开发者编码效率和代码质量的需求。该项目由 Chiikawa 组负责开发和维护，旨在提供一个集成本地模型和大语言模型 API 的 VSCode 插件，同时配备功能完善的 Web 前端，以满足不同用户的需求。

1.3 **定义**

- **ChiicaCode**：本项目名称，包括 VSCode 插件和 Web 前端两部分。
- **VSCode 插件**：Visual Studio Code 的扩展程序，提供代码辅助和聊天功能。
- **RAG（Retrieval-Augmented Generation）**：检索增强生成，用于项目级别代码生成的方法。
- **本地模型**：在用户本地环境中运行的 AI 模型，实现离线代码辅助功能。
- **大语言模型 API**：在线的 AI 模型接口，提供更丰富的代码辅助功能。

**2. 软件概述**

2.1 **目标**

ChiicaCode 的目标是为开发者提供一个智能、高效的代码辅助工具，集成代码解释、翻译、注释、自动补全等功能。通过支持本地模型和大语言模型 API，实现离线和在线两种模式的代码辅助。同时，利用 RAG 方法，提供项目级别的代码生成功能，提高开发效率。

2.2 **功能**

- **代码解释**：对选中的代码片段进行详细解释，帮助理解代码逻辑。
- **代码翻译**：将代码转换为其他编程语言，方便跨语言开发。
- **代码注释**：自动为代码添加注释，提高代码可读性。
- **自动补全**：智能预测并补全代码，提高编码速度。
- **项目级别代码生成**：通过 RAG 方法，根据项目需求自动生成代码结构。
- **Web 前端支持**：提供 Web 界面，支持上述所有功能，方便不使用 VSCode 的用户。

2.3 **性能**

**a. 数据精确度**

- **输入**：对用户输入的代码和指令进行高精度解析。
- **输出**：生成的代码和解释具有高准确性，符合编程规范。
- **处理数据**：在处理过程中，确保数据的完整性和准确性。

**b. 时间特性**

- **响应时间**：一般情况下，响应时间在 1 秒以内。
- **处理时间**：复杂的代码生成可能需要 2-3 秒，但不会超过 5 秒。
- **数据传输时间**：使用大语言模型 API 时，取决于网络状况，一般在 500 毫秒以内。

**c. 灵活性**

- **操作方式**：支持快捷键、命令面板和右键菜单等多种操作方式。
- **运行环境**：兼容 Windows、macOS 和 Linux 操作系统。
- **扩展性**：支持自定义模型和 API，满足特定需求。

**3. 运行环境**

3.1 **硬件**

- **计算机型号**：支持 x86 和 ARM 架构的计算机。
- **主存容量**：至少 8GB RAM，推荐 16GB 以上。
- **外存储器**：
  - **硬盘空间**：至少 20GB 可用空间。
  - **媒体**：SSD 或 HDD 均可。
- **输入/输出设备**：标准键盘、鼠标和显示器。
- **网络设备**：用于访问大语言模型 API，需要稳定的网络连接。

3.2 **支持软件**

- **操作系统**：
  - **Windows**：Windows 10 及以上版本。
  - **macOS**：macOS 10.15 及以上版本。
  - **Linux**：Ubuntu 18.04 及以上版本，或其他主流发行版。
- **VSCode**：Visual Studio Code 版本 1.50 及以上。
- **语言编译系统**：
  - **Node.js**：用于插件运行，版本 12 及以上。
  - **Python**：用于本地模型支持，版本 3.7 及以上。
- **其他必要的支持软件**：
  - **Git**：版本控制系统。
  - **NPM 或 Yarn**：包管理工具。

**4. 使用说明**

4.1 **安装和初始化**

- **VSCode 插件安装**：
  - 在 VSCode 扩展市场搜索 “ChiicaCode”。
  - 点击 “安装” 按钮，等待安装完成。
  - 安装完成后，重启 VSCode。
- **本地模型配置**：
  - 下载本地模型文件，存放于指定目录。
  - 在插件设置中，配置本地模型的路径。
- **API 设置**：
  - 在插件设置中，输入大语言模型 API 的密钥和终端地址。
- **反馈信息**：
  - 安装过程中，如果出现错误，插件会在输出面板显示错误信息。
  - 安装完成后，可以通过测试命令验证安装是否成功。

4.2 **输入**

4.2.1 **数据背景**

- **数据来源**：用户在编辑器中输入的代码、指令或自然语言文本。
- **存储媒体**：内存，即时处理，不会永久存储用户输入。
- **出现频度**：根据用户的操作频率。
- **限制和质量管理**：输入内容应符合编程语言规范，避免语法错误。

4.2.2 **数据格式**

- **长度**：输入的代码行数不限，但过长的代码可能影响性能。
- **格式基准**：遵循 VSCode 的代码格式化标准。
- **标号**：代码中的函数、变量应有明确的命名。
- **顺序**：按照代码逻辑顺序输入。
- **分隔符**：使用空格、缩进和换行符分隔代码块。
- **词汇表**：使用标准的编程语言关键字和库函数。
- **省略和重复**：避免重复代码，必要时可使用占位符。
- **控制**：遵循编程语言的控制结构，如循环、条件语句等。

4.2.3 **输入举例**


```javascript
function add(a, b) {
  return a + b;
}
```

4.3 **输出**

4.3.1 **数据背景**

- **输出数据的去向**：直接显示在 VSCode 的编辑器或输出面板中。
- **使用频度**：与用户的输入操作相对应。
- **存放媒体**：内存，可选择性地保存为文件。
- **质量管理**：输出的代码经过验证，确保正确性和有效性。

4.3.2 **数据格式**

- **首部**：输出的标题或提示信息。
- **主体**：生成的代码、解释或注释内容。
- **尾部**：附加的建议或提示。

4.3.3 **举例**

**代码解释输出**：

```
这是一个名为 add 的函数，接受两个参数 a 和 b，返回它们的和。
```

4.4 **出错和恢复**

- **出错信息及其含义**：
  - **网络错误**：无法连接到 API 服务，请检查网络连接。
  - **模型加载错误**：本地模型文件缺失或损坏。
  - **无效输入**：输入的代码存在语法错误或不支持的语言。
- **用户应采取的措施**：
  - 检查网络状况，确保能够访问互联网。
  - 确认本地模型文件的完整性和路径配置。
  - 修正输入的代码，使其符合语法规范。

4.5 **求助查询**

- **操作方法**：
  - 在插件设置中，点击 “帮助” 链接，访问在线文档。
  - 访问 ChiicaCode 的 GitHub 仓库，查看常见问题解答。
  - 通过邮件或社区论坛联系开发团队。

**5. 运行说明**

5.1 **运行表**

| 运行情况             | 运行目的                       |
| -------------------- | ------------------------------ |
| 初次安装后启动插件   | 初始化插件，开始使用功能       |
| 使用本地模型进行辅助 | 离线模式下提供代码辅助         |
| 使用 API 进行辅助    | 在线模式下提供更丰富的功能     |
| 启动 Web 前端        | 在浏览器中使用 ChiicaCode 功能 |

5.2 **运行步骤**

5.2.1 **运行控制**

- **无需特殊的运行控制**，插件会在 VSCode 启动时自动加载。

5.2.2 **操作信息**

- **运行目的**：提供代码辅助和项目级别代码生成功能。
- **操作要求**：正确安装插件，并配置必要的参数。
- **启动方法**：打开 VSCode，插件自动启动。
- **预计运行时间**：实时响应，无需等待。
- **操作命令格式及格式说明**：
  - 使用快捷键或命令面板调用功能，例如 `Ctrl+Shift+P` 打开命令面板，输入 `ChiicaCode`。
- **其他事项**：确保网络连接稳定，以获得最佳体验。

5.2.3 **输入/输出文件**

- **文件名称及编号**：用户项目中的代码文件，无特殊要求。
- **记录媒体**：硬盘或 SSD。
- **存留的目录**：用户项目所在的工作目录。
- **文件的支配**：由用户自行管理，插件不会自动修改文件内容。

5.2.4 **启动或恢复过程**

- **插件异常关闭**：重启 VSCode 或在插件管理中重新启用插件。
- **恢复默认设置**：在插件设置中选择 “恢复默认”，重置所有配置。

**6. 非常规过程**

- **出错处理操作**：
  - 当插件无法正常工作时，检查输出面板的错误日志。
  - 如果问题仍未解决，尝试重新安装插件。
- **切换到后备系统**：
  - 当 API 无法访问时，自动切换到本地模型。
  - 用户也可以在设置中手动选择使用本地模型或 API。
- **维护人员须知的操作和注意事项**：
  - 定期更新插件和模型，以获得最新功能和性能改进。
  - 备份配置文件，防止数据丢失。

  - **命令**：`ChiicaCode: Explain Code`
  - **功能**：对选中的代码进行解释。
- **代码翻译**：
  - **命令**：`ChiicaCode: Translate Code`
  - **功能**：将代码翻译为指定的编程语言。
- **添加注释**：
  - **命令**：`ChiicaCode: Add Comments`
  - **功能**：为代码自动添加注释。
- **自动补全**：
  - **命令**：自动触发，无需手动命令。
  - **功能**：根据上下文智能补全代码。

**8. 程序文件和数据文件一览表**

**9. 用户操作举例**

- **示例 1：使用代码解释功能**

  1. 在 VSCode 中打开一个 JavaScript 文件。
  2. 选中需要解释的代码片段。
  3. 按下快捷键 `Ctrl+Shift+P`，输入 `ChiicaCode: Explain Code`，按回车。
  4. 在输出面板查看代码解释。

- **示例 2：使用代码翻译功能**

  1. 选中一段 Python 代码。
  2. 按下快捷键 `Ctrl+Shift+P`，输入 `ChiicaCode: Translate Code`，按回车。
  3. 选择目标语言，例如 Java。
  4. 在新文件或编辑器中查看翻译后的代码。

- **示例 3：使用项目级别代码生成**

  1. 在插件的侧边栏，点击 “项目代码生成”。
  2. 输入项目需求描述，例如 “创建一个用户登录系统”。
  3. 点击 “生成代码” 按钮。
  4. 在项目目录中查看生成的代码文件。
